# Android Native Phase 0 Implementation Plan — Skeleton + CI + First APK

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a buildable, sideloadable Android app skeleton at `artifacts/android-native/` that produces a green CI run uploading `notepad3-android-debug-apk`. The first APK shows nothing more than a "Notepad 3++" Compose Hello — but it proves the toolchain works end-to-end.

**Architecture:** Standard Android single-module project. Kotlin + Jetpack Compose. Gradle 8.10 with Kotlin DSL. Min SDK 30, target SDK 34, compile SDK 35. Manual gradle wrapper bootstrap; CI on `ubuntu-latest` with Temurin JDK 17. No tests in this phase — the proof is "the APK builds, installs, and renders the hello text."

**Tech Stack:** Gradle 8.10, AGP 8.7.0, Kotlin 2.1.0, Compose BOM 2024.12.01, Compose Compiler Gradle Plugin (built-in via Kotlin 2.0+), AndroidX core-ktx + activity-compose + lifecycle-runtime-ktx, Material3.

### Task 1: Verify (or install) Temurin JDK 17 locally

AGP 8.7+ supports JDK 17 and 21. The CI workflow uses JDK 17, and the safest local match is JDK 17 too — JDK 21 sometimes surfaces deprecation warnings during AGP build classpath compilation. The user's machine already has Temurin 21 + Zulu 8, but no 17. Install it.

- [ ] **Step 2: Install Temurin 17 via Homebrew**

```

Expected: brew downloads the cask, installs into `/Library/Java/JavaVirtualMachines/temurin-17.jdk`.

- [ ] **Step 3: Verify the install**

```bash
/usr/libexec/java_home -v 17
```

Expected output: `/Library/Java/JavaVirtualMachines/temurin-17.jdk/Contents/Home`

- [ ] **Step 4: Export JAVA_HOME for the rest of this session**

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
"$JAVA_HOME/bin/java" -version

Expected output: `openjdk version "17.0.x"`. Use `"$JAVA_HOME/bin/gradle"` and `"$JAVA_HOME/bin/java"` explicitly going forward, or prepend `"$JAVA_HOME/bin"` to PATH for this shell.

**Why explicit invocation:** the existing `gradle` from `/opt/homebrew/bin/gradle` is currently broken on this machine because the `java` it picks up by default is corrupt. We sidestep that by always invoking via JAVA_HOME.

- [ ] **Step 1: Write the manifest**

```xml
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/Theme.Notepad3"
        tools:targetApi="34">

        <activity
            android:name=".MainActivity"
            android:exported="true"
            android:label="@string/app_name"
            android:configChanges="uiMode|orientation|screenSize|smallestScreenSize|screenLayout|keyboard|keyboardHidden|navigation"
            android:theme="@style/Theme.Notepad3">

            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>

</manifest>
```

**Why all those `android:configChanges` flags:** the spec architects for Activity-survival across config changes. Listing these tells Android *not* to recreate `MainActivity` on rotation, dark-mode flip, IME visibility, etc. — instead `onConfigurationChanged` is invoked and we react via `ThemeController.updateSystemStyle`. (This is the iOS-mental-model bridge; without these flags Android destroys + recreates the Activity, and ViewModels are the only thing that survives.)

# Notepad 3++ — Native Android Port

Kotlin / Jetpack Compose implementation of Notepad 3++ for Android. Sibling to the iOS Swift port at `../ios-native/`. The iOS app is the *behavioral oracle* — every design decision here matches iOS unless platform constraints force a substitution.

See `docs/superpowers/specs/2026-04-26-android-native-port-design.md` for the full design.

## Status

**Phase 0** — Skeleton + CI. The current APK launches and shows a Compose hello screen. Real features land in subsequent phases per the spec.

## Local build

Requires:
- macOS or Linux
- JDK 17 (`brew install --cask temurin@17` on macOS)
- Android SDK with build-tools 34+ and platform-tools (Android Studio installs these by default; cmdline-tools alone also works)
- Set `ANDROID_SDK_ROOT` (or `ANDROID_HOME`) to your SDK location

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)   # macOS
export ANDROID_SDK_ROOT="$HOME/Library/Android/sdk" # macOS
./gradlew :app:assembleDebug
# → app/build/outputs/apk/debug/app-debug.apk

## CI build

Push a tag matching `android-native-v*` (e.g. `android-native-v0.1.0`) or run the workflow manually:

```bash
gh workflow run "Build Android Native" --ref <branch>
gh run watch --exit-status
gh run download <run-id> -n notepad3-android-debug-apk
```

Artifact: `notepad3-android-debug-apk` containing `app-debug.apk`.

## Sideloading

1. Enable Developer Options on your Android device (Settings → About phone → tap Build number 7 times).
2. Enable USB Debugging.
3. Connect via USB, then `adb install path/to/app-debug.apk`.

Or, transfer the APK to the device (cloud, AirDrop-equivalent, etc.) and tap to install. Android will prompt to allow installs from this source.

### Task 11: CI workflow

**Files:**
- Create: `.github/workflows/build-android-native.yml`

- [ ] **Step 1: Write the workflow**

```yaml
name: Build Android Native

on:
  workflow_dispatch:
  push:
    tags:
      - "android-native-v*"

jobs:
  build:
    name: Build debug APK (Kotlin / Compose)
    runs-on: ubuntu-latest
    timeout-minutes: 30

    defaults:
      run:
        working-directory: artifacts/android-native

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Java (Temurin 17)
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: "17"

      - name: Setup Gradle
        uses: gradle/actions/setup-gradle@v4

      - name: Show toolchain versions
        run: |
          java -version
          ./gradlew --version

      - name: Build debug APK
        shell: bash
        run: |
          set -euo pipefail
          chmod +x ./gradlew
          ./gradlew :app:assembleDebug --no-daemon --stacktrace
          echo "==> APK outputs:"
          find app/build/outputs/apk -name "*.apk" -print

      - name: Upload debug APK
        uses: actions/upload-artifact@v4
        with:
          name: notepad3-android-debug-apk
          path: artifacts/android-native/app/build/outputs/apk/debug/*.apk
          if-no-files-found: error
```
